# 16-bit Ripple Carry Adder (RCA)

## Overview

This project implements a **16-bit Ripple Carry Adder (RCA)** in Verilog HDL. The design performs binary addition of two 16-bit operands using a chain of sixteen **1-bit Full Adders**. Each Full Adder generates a carry that propagates to the next stage, giving the architecture its name.

The Ripple Carry Adder is one of the simplest arithmetic circuits and serves as a fundamental building block for Arithmetic Logic Units (ALUs) and other digital systems. While easy to design and understand, its carry propagation delay increases linearly with the operand width.

## Theory

A **Full Adder** adds three one-bit inputs:

* **A** – First input bit
* **B** – Second input bit
* **Cin** – Carry input from the previous stage

It produces:

* **Sum** – Sum bit
* **Cout** – Carry output

A 16-bit Ripple Carry Adder is formed by cascading sixteen Full Adders. The carry generated by each stage becomes the carry input of the next stage until the final carry-out is produced.

```
Cin
 │
 ▼
FA0 → FA1 → FA2 → ... → FA15
 │     │      │            │
 ▼     ▼      ▼            ▼
S0    S1     S2          S15

Final Carry-Out
```

## Limitations

The Ripple Carry Adder has a propagation delay proportional to the number of bits because each Full Adder must wait for the carry from the previous stage. For larger operand widths, faster architectures such as Carry Lookahead Adders (CLA), Carry Select Adders (CSLA), or Parallel Prefix Adders are generally preferred.
